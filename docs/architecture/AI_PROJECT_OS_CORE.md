# AI 项目推进操作系统核心架构

## 1. 架构决议

采用“标准版 + 自研薄内核 + 可选配置 + 领域包 + 项目实例 + 开源适配器”。

```text
L1 AI Project OS
  → 提供通用协议、状态、门禁、模板、检查和研究方法

L2 业务系统
  → 提供业务事实、场景、链路、契约、工作流和业务技能

L3 项目实例
  → 提供具体项目事实、配置、数据、运行、证据和交付物
```

依赖只能由 L3 指向 L2、由 L2 指向 L1。L1 不得引用任何具体业务仓库。

## 2. 薄内核六个原语

1. 项目目标与范围。
2. 受控产物及其权威位置。
3. 稳定编号与有类型关系。
4. 状态迁移及重新开启条件。
5. 阶段门禁与完成声明规则。
6. 运行记录与验证证据。

以下能力不是标准版强制内核：图数据库、独立注册中心、完整工作流平台、上下文服务、企业资产门户、通用补偿引擎。

## 3. 五个架构平面

| 平面 | 职责 |
|---|---|
| 控制平面 | 阶段、状态、门禁、评分和声明封顶 |
| 认知平面 | 来源、事实、假设、未知、研究和决策 |
| 设计平面 | 场景、触发、业务链路、能力、功能、架构和契约 |
| 执行平面 | 规格、任务、工作流、技能、工具、检查点和副作用 |
| 证据平面 | 测试、运行结果、证据等级、复盘、升格和失效 |

五个平面共享稳定编号和追溯关系，不共享一份无限增长的超级文档。

## 4. 正确的主依赖方向

```text
原始来源
→ 批准事实和需求
→ 场景与触发档案
→ 业务链路
→ 能力树和功能树
→ 架构与工程链路
→ 输入输出契约和验收判据
→ 规格控制基线
→ 任务依赖图
→ 工作流
→ 技能与工具
→ 运行
→ 证据
→ 完成声明
```

下游失败可以请求上游复审，但不能直接反向修改上游事实。上游变更会把相关下游资产标记为待复核或过期。

## 5. 标准版范围

标准版必须包含：

- R0、S0—S7 生命周期；
- 领域和技术研究；
- 架构决策记录；
- 场景、触发和链路包；
- 输入输出语义契约；
- 风险自适应规格；
- 任务、工作流和技能边界；
- 追溯检查；
- 分层证据和声明封顶；
- 最小模拟运行和本地运行证明。

标准版不包含：

- 企业项目组合管理；
- 多租户身份平台；
- 自研持久工作流引擎；
- 自研 AI 可观测平台；
- 全部开源候选的默认安装；
- 未经研究的生产依赖。

## 6. 开源复用边界

外部能力只通过适配器接入：

| 能力 | 候选 |
|---|---|
| 规格驱动 | Spec Kit、OpenSpec、Spec Kitty、BMAD |
| 架构即代码 | Structurizr DSL |
| 技术研究 | GPT Researcher |
| 智能体状态机 | LangGraph |
| 生产持久流程 | Temporal |
| 观测和评测 | OpenTelemetry、Phoenix、Langfuse |
| 策略即代码 | Open Policy Agent |

任何候选都必须先进入 `research/`，完成资料审计、本地实验、退出成本分析和架构决策后才能成为依赖。
