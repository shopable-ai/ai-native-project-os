# 审核策略认证

| 阅读契约 | 内容 |
|---|---|
| 解决的问题 | 在规则、Prompt 或 AI reviewer 驱动自动审核前，怎样证明整套审核策略在正例、反例、边界和对抗输入上达到预注册要求。 |
| 何时阅读 | 新建或修改审核规则、Prompt、模型、输出 Schema、Context 策略、审核阈值或自动激活范围时。 |
| 输入 | 候选审核策略包、风险路由、版本化测试集、指标与阈值、Run 和 Evidence。 |
| 输出 | 审核策略认证 Verdict、允许的批准路由、声明上限、失效条件和重开目标。 |
| 下一步 | 认证通过后按激活 Policy 进入自动策略决策或人工确认；失败、缺口或失效时保持 blocked 并重开对应设计或测试。 |

本文是“审核规则和审核推理怎样在自动运行前获得可信资格”的人类操作权威。机器结构分别由审核策略测试集 Contract 与审核策略认证 Contract 定义；自动激活、人工确认和阻断条件由激活 Policy 定义。本文不复制完整机器字段或项目专用阈值。

## 1. 人工治理不等于逐条人工审核

人类负责确定目标、责任、风险边界、不可自动放宽的条件和高风险授权。普通内容审核、满足预先授权条件的低风险需求以及不降低控制强度的小版本规则，可以由已经认证的策略自动裁决。

```text
候选对象
→ 决策门（Decision Gate）
   ├─ policy_certified：有效认证与路由 Policy 允许，自动裁决
   ├─ human_signoff：治理边界或高风险变化，人工确认
   └─ blocked：证据不足、规则缺口或测试失败
```

`approval_status` 继续表示对象当前审批结果；批准路由只解释该结果由什么权威路径产生。二者不能合并成新状态轴。

## 2. 审核策略包

认证对象必须是完整审核策略包，而不是孤立 Prompt：

- active 规则集的稳定 ID、版本和 hash；
- Prompt 模板与实际装配规则的版本/hash；
- 输入和输出 Schema；
- Context 选择、信任、排序、截断与排除策略；
- 模型 fingerprint、参数和允许的 fallback；
- Tool、权限、项目 namespace 和数据边界；
- reviewer 独立性、改写次数、失败返回和 Rule Gap 出口。

Prompt 负责装配规则，不是规则权威源。模型、Context 或 Schema 变化即使 Prompt 文本未变，也属于新的策略包版本。

## 3. 预注册测试

测试必须在执行前固定 case 输入或输入引用、预期 decision、预期规则引用、禁止结果、必需 finding/Evidence、风险级别和重复次数。最低类别包括：

- 正例、反例、边界例和对抗例；
- Unknown、规则冲突和过期规则；
- 跨项目隔离和适用时的多语言一致性；
- 有界改写、达到上限后的阻断和新规则恢复重审。

不得在观察模型输出后修改预期结果，也不得只保留表现最佳的尝试。

## 4. 多轮测试和指标

确定性结构 gate 可按版本执行一次；非确定性 AI reviewer 必须按测试集声明的次数重复运行。每次尝试保存模型与参数指纹、输入 hash、实际 Context、输出、规则引用、decision、finding、耗时与失败。

最低观测包括误放行率、误阻断率、决策稳定度、规则引用准确率、Schema 有效率、规则缺口识别率、改写成功率和跨项目泄漏率。阈值由项目 Policy 预注册，高风险对象优先限制误放行，不能只用平均准确率掩盖关键失败。

## 5. 独立认证

策略生成者、运行 reviewer 与认证 verifier 必须按风险路由满足职责分离。AI 可以执行测试和生成候选认证报告，但 AI 不能给自己的策略签发独立认证，也不能修改激活 Policy、降低阈值或扩大 scope 后继续沿用旧认证。

认证 Verdict 至少说明：测试包和策略包版本/hash、实际 Run/Evidence、指标结果、未覆盖范围、有效期、声明上限、失效条件和认证主体。认证通过不授予外部动作权限。

## 6. 风险路由

允许自动激活的典型范围：不扩大 scope、不降低阈值、不删除 blocking 规则、不增加权限，并且完整回归通过的低风险变化。

必须人工确认的典型范围：原始目标或责任变化、新规则族、适用范围扩大、审核阈值降低、风险接受、删除阻断规则、权限扩大和未知外部副作用。

不可逆发送、付款、删除和生产发布继续要求独立动作授权；内容或需求通过决策门不能替代授权快照。

## 7. 失效与重开

规则、Prompt、模型/参数、Schema、Context 策略、Tool 权限、测试集、指标、阈值、项目 scope 或 reviewer 独立性变化时，旧认证进入待复核或失效。新反例、事故和真实业务结果也可以推翻旧认证。

旧 Run、Evidence 和 Verdict 保持不可改写；使用旧认证的 Baseline、Spec、Workflow、审核 Verdict 和 Claim 沿依赖关系失效，并重开最早受影响阶段。

## 8. 当前证明边界

本仓当前只准备建立权威、Contract、Policy、模板、检查器和 synthetic fixture。没有真实模型多轮 Run、真实 L2 消费、跨项目隔离或生产运行 Evidence 时，不得声明审核运行时就绪、真实稳定性或通用 95+。
