# 项目生命周期：R0 与 S0—S7

## 1. 总流程

```text
R0 存量架构恢复（仅存量项目）
→ S0 目标与边界
→ S1 领域和技术认知
→ S2 场景与业务链路
→ S3 方案和架构决策
→ S4 工程设计
→ S5 计划与最小执行
→ S6 验证与发布
→ S7 运行与演进
```

阶段是成熟度坐标，不是文件目录，也不是只能向前的瀑布流程。

## 2. 阶段定义

| 阶段 | 核心输入 | 必需输出 | 退出门禁 | 常见重开条件 |
|---|---|---|---|---|
| R0 存量恢复 | 现有仓库、文档、运行资产 | 当前架构、需求反向映射、差距、迁移计划 | 关键实现均有来源或被标记为孤儿 | 发现新的活动入口或事实源 |
| S0 目标边界 | 需求种子 | 目标、非目标、角色、成功指标、项目类型初判 | 目标负责人和验收方向明确 | 商业目标或范围变化 |
| S1 认知研究 | 目标、来源、未知项 | 来源清单、事实/假设/未知、术语、研究结论 | 关键未知不再阻断当前决策 | 新来源、法规或技术前提变化 |
| S2 场景链路 | 批准需求、领域事实 | 场景、触发档案、正常/异常/恢复链路 | P0/P1 需求均有场景和链路 | 用户、触发或业务政策变化 |
| S3 架构决策 | 链路、质量属性、约束 | 候选方案、评分、反方报告、决策记录 | 关键选择有证据、后果和退出条件 | 关键约束或候选能力变化 |
| S4 工程设计 | 已接受架构 | 能力、工程链路、I/O、契约、验收判据 | 接缝、失败态、权限和兼容完整 | 契约、数据或外部接口变化 |
| S5 计划执行 | 工程设计、验收判据 | 规格、任务图、工作流、技能、最小纵向实现 | 工作包可执行、可暂停、可恢复 | 计划失败、依赖阻断或实现反证 |
| S6 验证发布 | 实现和测试计划 | 测试报告、运行结果、证据包、发布决策 | 结构、语义、追溯和证据等级全部通过 | 测试失败、证据过期或发布拒绝 |
| S7 运行演进 | 发布版本和真实运行 | 指标、事故、复盘、升降级和废弃记录 | 真实结果、成本、责任和恢复链成立 | 生产事故、效果不达标或新目标 |

## 3. 四个正交状态

每个工作项分别记录：

```text
生命周期位置：R0 / S0 ... S7
工作状态：排队 / 执行 / 等待人工 / 阻断 / 暂停 / 失败 / 完成 / 被替代
证据等级：控制包 / 契约 / 模拟 / 本地 / 只读真实 / 生产
审批状态：不需要 / 待审批 / 已批准 / 拒绝 / 过期 / 撤销
```

禁止用一个 `completed` 同时表达四种状态。

## 4. 门禁执行规则

```text
门禁通过 =
  命令退出码为 0
  且必需产物存在
  且结构校验通过
  且语义不变量通过
  且 P0/P1 问题为 0
  且关键追溯覆盖率为 100%
  且证据未过期
  且证据等级达到声明要求
```

低风险项目允许带责任人和有效期的显式豁免。安全、隐私、外部不可逆副作用和生产声明不允许静默豁免。

## 5. 允许的回退

```text
S6 验收失败 → S4 修正契约或 S5 修正实现
S6 发现目标错误 → S0 重新确认
S7 发现业务假设错误 → S1 或 S2 重开
S7 外部接口变化 → S4 契约失效
S7 事故证明架构错误 → S3 新建替代决策
```
