# R0 与 S0—S7 阶段退出门禁

| 阅读契约 | 内容 |
|---|---|
| 解决的问题 | 给人类逐阶段说明什么输入和产物足以申请退出、什么 Evidence 能支撑裁决，以及失效后重开哪里。 |
| 何时阅读 | 阶段开始、阶段退出审查、Evidence 过期或上游变化时。 |
| 输入 | 当前阶段、批准基线、必需产物、验证命令与 Evidence。 |
| 输出 | 阶段退出决定、缺口、失效影响和重开目标。 |
| 下一步 | 门禁通过后进入[项目交付工作流](PROJECT_DELIVERY_WORKFLOW.md)的下一个依赖节点；失败时按本文件指定目标重开。 |

本文件是阶段门禁的人类操作视图。[项目生命周期](PROJECT_LIFECYCLE.md)定义阶段语义，[门禁、证据等级与评分](../governance/GATES_PROOF_SCORING.md)定义唯一证据枚举与通用公式，[状态迁移与失效传播](../governance/STATE_TRANSITIONS_AND_INVALIDATION.md)定义状态和失效。本文不复制这些机器枚举。

## R0 — 存量架构恢复

- **输入**：`{{l2_repo}}` 的活动代码、配置、文档、运行入口、历史迁移材料和可读取 Evidence。
- **产物**：当前架构图、入口与依赖清单、批准需求反向映射、孤儿实现清单、差距与迁移计划。
- **退出门禁**：关键活动实现都有可审计来源，或被明确标记为孤儿并进入处置；真实入口和未知项有 owner。
- **Evidence**：仓库/构建/只读运行清点、来源引用、hash 和复现命令；其证明边界由 Evidence 自身声明。
- **失效条件**：发现新入口、新事实源、隐藏依赖或现状扫描范围变化。
- **重开目标**：重开 R0；若推翻目标或需求，同时重开 S0/S1。

## S0 — 目标与边界

- **输入**：需求种子或 R0 恢复结果、利益相关方、约束和已知非目标。
- **产物**：目标、非目标、角色、成功指标、需求基线、项目类型初判与治理路由初版。
- **退出门禁**：目标 owner、范围分母、验收方向和禁止声明明确；未知输入不被默认为低风险。
- **Evidence**：批准记录、需求基线 hash、范围差异与路由决策引用。
- **失效条件**：商业目标、范围、责任人、项目类型或风险输入变化。
- **重开目标**：重开 S0，并使依赖旧基线的 Spec、Task、Evidence、Verdict、Claim 待复核。

## S1 — 领域与技术认知

- **输入**：目标、批准来源、假设、未知项和待决问题。
- **产物**：来源清单、术语、批准事实、假设/未知登记、研究包、研究 Verdict 或候选 ADR。
- **退出门禁**：关键未知不再阻断当前决策；来源没有绕过批准事实直接驱动实现。
- **Evidence**：来源审计、研究命令/实验、反方审查、引用和研究裁决。
- **失效条件**：新来源、法规、领域事实、技术前提或研究输入变化。
- **重开目标**：重开 S1；若事实影响目标或场景，联动 S0/S2。

## S2 — 场景与业务链路

- **输入**：批准需求、领域事实、触发条件和角色。
- **产物**：场景、触发档案、业务链路、正常/异常/恢复路径、能力树和功能树。
- **退出门禁**：P0/P1 需求有场景、触发、链路、失败终态与恢复出口，业务能力不预绑工程实现。
- **Evidence**：需求到场景/链路覆盖、分支与终态检查、责任人审查记录。
- **失效条件**：用户、触发、业务政策、关键事实或需求分母变化。
- **重开目标**：重开 S2；目标被推翻时回 S0，事实被推翻时回 S1。

## S3 — 方案与架构决策

- **输入**：业务链路、质量属性、约束、研究 Verdict 和候选方案。
- **产物**：方案比较、反方报告、已接受/拒绝 ADR、后果、退出条件和替代关系。
- **退出门禁**：关键选择有可重算依据，风险、代价、退出与回退条件明确，未决研究只阻断具体决策。
- **Evidence**：候选评分输入、实验/研究 Evidence、独立反方意见和 ADR 批准记录。
- **失效条件**：关键约束、候选能力、风险阈值或上游业务链变化。
- **重开目标**：重开 S3；需要新认知时回 S1，需要重画业务链时回 S2。

## S4 — 工程设计

- **输入**：已接受 ADR、业务能力、质量属性和边界约束。
- **产物**：工程链路、组件职责、I/O 与失败契约、权限/隔离设计、验收判据和兼容策略。
- **退出门禁**：所有接缝、失败态、未知副作用、权限、数据分类、恢复和兼容条件有明确 owner 与验收出口。
- **Evidence**：Schema/契约解析、正反例契约测试设计、威胁/失败路径审查和图覆盖记录。
- **失效条件**：ADR、契约、数据、权限、外部接口或兼容范围变化。
- **重开目标**：重开 S4；架构假设错误时回 S3。

## S5 — 计划与最小执行

- **输入**：工程设计、验收判据、批准 Spec 基线、依赖和授权约束。
- **产物**：Spec 包、从 Spec 推导的任务树、Workflow、局部 Skill/Tool、最小纵向实现和恢复计划。
- **退出门禁**：工作包可执行、可暂停、可恢复；Task 无孤儿，Workflow 编排 Task，Skill/Tool 不隐藏控制流或声明权限。
- **Evidence**：静态实现检查、契约测试、Task 覆盖、依赖/授权校验和恢复演练计划；静态通过不自动提升为运行证明。
- **失效条件**：Spec、判据、依赖、实现反证、权限或执行计划变化。
- **重开目标**：实现或计划问题重开 S5；契约问题回 S4；方案问题回 S3。

## S6 — 验证与发布

- **输入**：实现、测试计划、事前判据、授权快照和候选发布版本。
- **产物**：Run、Evidence 包、Acceptance Verdict、发布/拒绝决定、失败与恢复记录、受限 Claim。
- **退出门禁**：结构、语义、追溯、职责分离、新鲜度和声明所需证明边界全部通过；未知副作用保持阻断。
- **Evidence**：不可改写 Run、逐判据 Evidence、独立 Verdict、环境/内容指纹、未覆盖范围和复现命令。
- **失效条件**：测试失败、Evidence 过期、Verdict 撤销、发布拒绝、版本或环境指纹变化。
- **重开目标**：实现失败回 S5，契约失败回 S4，架构错误回 S3，目标错误回 S0。

## S7 — 运行与演进

- **输入**：经批准发布版本、真实运行、运营指标、事故和用户/业务反馈。
- **产物**：生产 Run/Evidence、效果与成本记录、事故/恢复、复盘、资产升降级或废弃提案。
- **退出门禁**：真实结果、责任、恢复与残余风险可审计；资产升格满足跨项目和异构验证门禁。
- **Evidence**：生产环境指纹、外部回执、指标、事故与恢复 Evidence、独立 Verdict；不得用 fixture 或本地运行替代。
- **失效条件**：生产事故、效果不达标、新目标、外部接口/法规变化或业务假设被反证。
- **重开目标**：假设错误回 S1/S2，架构错误回 S3，接口/契约变化回 S4，实现问题回 S5。
