# 领域与技术研究工作流

## 1. 研究触发条件

出现以下任一情况时创建研究任务：

- 执行者不熟悉的概念位于关键路径；
- 存在两个以上合理方案；
- 失败影响、迁移成本或不可逆性较高；
- 涉及外部框架、接口、模型、数据、安全、隐私或合规；
- 关键判断只来自 AI 记忆，没有当前权威来源；
- 性能、成本或可靠性假设没有实验支持。

不研究与当前决策无关的普通关键词。

## 2. 研究优先级

```text
priority = impact × uncertainty × irreversibility
```

三个维度按 1—5 评分。任一维度为 5，或总分不低于 48，并且研究位于当前关键路径时，阻断相关架构决策。

## 3. 研究流程

```text
发现未知或风险
→ 定义研究问题和决策边界
→ 搜索官方资料、原始论文和开源项目
→ 许可证、安全、活跃度和可替换性初筛
→ 形成至少两个候选
→ 设计最小实验
→ 收集证据
→ 反方攻击
→ 采用 / 适配 / 延后 / 拒绝
→ 形成架构决策
→ 更新契约、规格和任务
```

## 4. 研究包必需内容

| 文件或目录 | 职责 |
|---|---|
| `brief.md` | 问题、范围、阻断阶段、成功条件 |
| `sources.yaml` | 来源、权威等级、日期、适用范围 |
| `options.md` | 候选、评分和淘汰原因 |
| `experiment-plan.md` | 最小实验、数据、命令和判据 |
| `evidence/` | 原始输出、哈希、环境和结果 |
| `adversarial-review.md` | 反方攻击、失败条件和剩余风险 |
| `verdict.md` | 决策、置信度、适用边界、重审条件 |

## 5. 研究结果的升格

```text
确认的业务事实 → 业务系统的 business/
架构选择       → decisions/
稳定接口       → contracts/
工程控制要求   → specs/
可复用编排     → workflows/
成熟局部能力   → skills/
原始研究材料   → research/archive/
```

研究报告本身不得直接成为业务事实或生产声明。

## 6. 开源采用门禁

任何第三方候选必须检查：

- 许可证和分发限制；
- 维护活跃度和负责人；
- 安全记录和供应链；
- 数据是否外传；
- 离线和自托管能力；
- 与核心协议的适配成本；
- 版本锁定和升级路径；
- 导出、替换和退出成本；
- 本地最小实验；
- 不采用时的后备方案。
