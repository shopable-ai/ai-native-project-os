# 内容与证据审核规则

## review.content.quality_and_clarity

```yaml
rule_id: review.content.quality_and_clarity
version: 1
priority: 60
canonical_path: governance/rules/内容与证据审核规则.md#review.content.quality_and_clarity
severity: blocking
applies_to:
  artifact_classes: [generated_content]
required_evidence: [subject_content, project_context_manifest]
allowed_outcomes: [allow, rewrite_required, blocked, rule_gap]
failure_action: rewrite_then_block_at_limit
```

审核输出是否完成目标、结构清楚、没有互相矛盾的陈述，并与目标读者和已批准表达约束一致。可修复问题进入有界改写；缺少适用规则时登记规则缺口。

## review.evidence.claim_grounding

```yaml
rule_id: review.evidence.claim_grounding
version: 1
priority: 20
canonical_path: governance/rules/内容与证据审核规则.md#review.evidence.claim_grounding
severity: blocking
applies_to:
  artifact_classes: [generated_content, evidence, acceptance_input]
required_evidence: [claim_manifest, fact_refs, evidence_refs]
allowed_outcomes: [allow, rewrite_required, blocked, rule_gap]
failure_action: remove_or_ground_claim_then_rereview
```

每项可验证声明必须绑定可追溯事实或 Evidence。无法补充依据的声明必须删除、降低声明上限或阻断，不能用流畅措辞替代证据。
