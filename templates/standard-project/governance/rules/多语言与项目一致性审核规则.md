# 多语言与项目一致性审核规则

## review.locale.rule_conformance

```yaml
rule_id: review.locale.rule_conformance
version: 1
priority: 50
canonical_path: governance/rules/多语言与项目一致性审核规则.md#review.locale.rule_conformance
severity: blocking
applies_to:
  locale_source: structured_input
required_evidence: [locale, localized_rule_refs, subject_content]
allowed_outcomes: [allow, rewrite_required, blocked, rule_gap]
failure_action: rewrite_or_open_rule_gap
```

审核表达是否符合输入中结构化 `locale` 所选择的已批准规则，并保持含义、风险边界和声明上限一致。L1 不维护具体语言词面清单。

## review.project.fact_alignment

```yaml
rule_id: review.project.fact_alignment
version: 1
priority: 40
canonical_path: governance/rules/多语言与项目一致性审核规则.md#review.project.fact_alignment
severity: blocking
applies_to:
  governance_scope: l3
required_evidence: [project_id, project_fact_refs, context_manifest, subject_content]
allowed_outcomes: [allow, rewrite_required, blocked, rule_gap]
failure_action: rewrite_or_block
```

审核对象只能使用当前 `project_id` namespace 下批准的事实、规则和 Evidence，不得混入其他项目私有上下文。共享资产必须有版本化引用和授权边界。
