# 审核规则集说明

本目录保存由人类治理、经合法 Decision Gate 发布和维护的审核规则。Prompt 运行时只装配这里已经发布的规则版本，不得复制出另一份可编辑规则。

## 发布流程

1. 规则 owner 修改具体规则文件并形成新版本。
2. 计算每个成员文件和本说明的内容 hash。
3. 更新完整审核策略包 hash，在运行前固定正例、反例、边界、对抗、重复次数、指标和阈值。
4. 执行全部测试 Run，保存全部尝试与 Evidence，由独立 verifier 签发认证 Verdict。
5. 激活 Policy 选择 `policy_certified` 或 `human_signoff`；scope 扩大、阈值降低、阻断规则删除、权限增加、目标/责任变化和风险接受必须转人工。
6. 填写批准路由、决策权威、认证或人工签署证据以及生效时间，再将 `status` 改为 `active`。
7. 新版本通过后使旧版本 `superseded`；运行中发现的 `rule_gap_case` 由新规则、新认证和新审核 Run 复核后关闭。

## 运行约束

- AI 可以提出修订、执行测试和生成候选报告，但不能修改 active 规则、给自己的策略签发独立认证或改变激活 Policy 后沿用旧认证。
- 每项审核 finding 必须引用本清单中的精确 `rule_ref`。
- 同一 scope 内每条规则必须有唯一 `priority`；冲突无法按明确优先级解决时形成 `rule_gap` 并阻断。
- 规则缺失、过期、hash 不匹配、scope 不覆盖或优先级冲突时安全阻断。
- 文件名和正文服务人工阅读；机器只使用稳定 ID、结构化字段、版本和 hash，不按自然语言关键词分流。
