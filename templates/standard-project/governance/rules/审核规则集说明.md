# 审核规则集说明

本目录保存人类定义、批准、发布和维护的审核规则。Prompt 运行时只装配这里已经发布的规则版本，不得复制出另一份可编辑规则。

## 发布流程

1. 规则 owner 修改具体规则文件并形成新版本。
2. 计算每个成员文件和本说明的内容 hash。
3. 人类 approver 检查 scope、优先级、Evidence 要求、失败动作和兼容影响。
4. 填写 `approved_by/approved_at/effective_from`，将 `status` 改为 `active`。
5. 新版本通过后使旧版本 `superseded`；运行中发现的 `rule_gap_case` 由新审核 Run 复核后关闭。

## 运行约束

- AI 只能提出修订建议，不能修改 active 规则或填写人工批准身份。
- 每项审核 finding 必须引用本清单中的精确 `rule_ref`。
- 规则缺失、过期、hash 不匹配、scope 不覆盖或优先级冲突时安全阻断。
- 文件名和正文服务人工阅读；机器只使用稳定 ID、结构化字段、版本和 hash，不按自然语言关键词分流。
