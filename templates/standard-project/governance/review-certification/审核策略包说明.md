# 审核策略包说明

| 问题 | 何时填写 | 输入 | 输出 | 下一步 |
|---|---|---|---|---|
| 哪些内容共同决定 reviewer 的实际行为？ | 新建或修改规则、Prompt、Schema、Context、模型、Tool 或权限时 | 全部组件的精确版本与 hash | 一个不可混用的候选审核策略包 | 预注册测试集，再执行全部重复 Run |

本文件是 L2 可复制的候选策略包说明，不是已认证策略。复制后保持 `bundle_state: candidate`，直到全部组件、测试集、Run/Evidence 和独立认证 Verdict 可解析。

## 完整组件边界

- 规则集决定允许、改写、阻断和 Rule Gap 的规范；
- Prompt 只装配规则，不是规则权威；
- 输入/输出 Schema 固定可判定结构和失败返回；
- Context Policy 固定选择、信任、排序、截断和排除；
- 模型 fingerprint 包含实际模型、参数和 fallback 边界；
- Tool 与权限集合固定 reviewer 能读取和执行什么；
- 改写上限和失败出口固定终止条件。

任一组件变化都生成新的 `bundle_hash`，旧认证至少进入待复核。不得只修改 Prompt 文本后沿用旧认证。

## 职责分离

`generator_ref` 可以生成策略草稿和运行测试，但不能作为自己的独立认证 verifier。Verifier 必须检查全部尝试、指标分母、阈值、未覆盖范围和 hash 绑定；不能删除失败尝试来制造通过。

## 声明边界

本模板只证明需要收集哪些静态信息。占位符、fixture 或检查器通过都不证明真实模型稳定、真实 L2 可用、外部动作获权或生产安全。
